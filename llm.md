 Изначальный контекст проекта LummeCraft (Static HTML/CSS/JS)

Этот файл содержит описание оригинальной структуры, архитектуры и файлов проекта LummeCraft до его переноса на полноценный бэкенд Express + React. Используйте его для восстановления изначального контекста в новых сессиях или сторонних LLM для экономии лимитов токенов.

## 📁 Изначальная структура файлов статического сайта
Проект представлял собой многостраничный статический сайт (Multi-Page Application), работающий полностью на стороне клиента с прямой интеграцией сторонних API (Telegram, EmailJS) через браузер.

```text
/
├── index.html                                 # Главная страница (Промо-блоки, Хиты, Преимущества, FAQ)
├── catalog.html                               # Страница каталога (Сетка/список товаров, фильтрация, сортировка)
├── check-order.html                           # Страница проверки статуса заказа по ID и телефону
├── cart-modal.html                            # Шаблон модального окна корзины (динамически подгружаемый через fetch)
├── css/
│   ├── vars.css                               # Цветовая палитра, переменные, скругления, тени
│   ├── ui.css                                 # Базовые UI-компоненты (кнопки, поля ввода, карточки наборов)
│   ├── home.css                               # Стили главной страницы (Hero, сетка хитов, бегущая строка)
│   ├── home-extra.css                         # Анимации, боковые волны, адаптивность шапки
│   ├── catalog.css                            # Стили страницы каталога и фильтров
│   ├── cart-items.css                         # Оформление списка товаров и управления количеством в корзине
│   ├── transitions.css                        # Плавные переходы и анимация открытия модальных окон
│   └── modal-styles.css                       # Оформление карточки детального просмотра товара
└── js/
    ├── config.js                              # Конфигурационный файл (токен Telegram, Chat ID, EmailJS)
    ├── data.ts / data.js                      # База данных 16 оригинальных наборов и промокоды
    ├── api.js                                 # Клиент для REST API (методы авторизации, заказов, статусов)
    ├── shared.js                              # Общий код (шапка, меню, корзина, детальный просмотр товара)
    ├── cart-modal.js                          # Логика работы корзины, калькуляция цен, чекаут
    ├── catalog.js                             # Логика каталога (фильтрация по типу, размеру, цене, сложности)
    └── modal-logic.js                         # Интерактивное открытие детального просмотра ("The Artisan's Reveal")
```

## ⚙️ Описание изначальной архитектуры и логики

### 1. Браузерное хранилище (Client-Side State)
*   **Корзина (`cart`)** и **Избранное (`favorites`)** полностью сохранялись на стороне клиента в `localStorage` браузера (`lc_cart_v2`, `lc_favs_v2`).
*   Изменение количества товаров в корзине, применение промокодов и расчет итоговой стоимости происходили синхронно в браузере при помощи чистых JS-функций.

### 2. Прямые интеграции сторонних API из браузера
*   **Telegram-уведомления**: Заказ отправлялся напрямую из клиентского файла `notify.js` через публичный запрос `fetch('https://api.telegram.org/bot<TG_TOKEN>/sendMessage')` с использованием токена и Chat ID, указанных в `window.CONFIG` (`js/config.js`).
*   **EmailJS**: Уведомления на почту магазина отправлялись напрямую из браузера с помощью вызова `emailjs.send(service_id, template_id, params)` после инициализации публичного ключа в шапке.

### 3. Модальные окна и интерактивность
*   **Шаблон корзины**: Файл `cart-modal.html` загружался асинхронно через `fetch()` на страницах `index.html` и `catalog.html` для исключения дублирования разметки оформления заказа.
*   **Детальный просмотр**: Эффектное открытие карточки товара с помощью `clip-path: circle()` и плавных CSS-переходов. Автоматическая генерация состава набора в зависимости от категории (краски/кисти для картин, стилус/стразы для мозаики).
*   **Маска телефона**: Поле ввода номера телефона автоматически форматировало ввод под белорусский стандарт `+375 (XX) XXX-XX-XX`.

### 4. Динамический каталог и фильтрация
*   Клиентский скрипт `js/catalog.js` обеспечивал живой поиск по каталогу, мультивыбор категорий (Картины / Мозаика), размеров (30х40, 40х50, 50х65, 60х80), регулировку диапазона цен, фильтрацию по сложности и сортировку (по цене, популярности, рейтингу, размеру скидки).
